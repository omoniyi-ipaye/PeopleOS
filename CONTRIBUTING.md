# Contributing to PeopleOS

Thank you for your interest in PeopleOS! We welcome contributions from developers who want to help build the future of privacy-preserving People Analytics.

## How to Contribute

1.  **Fork the Repository**: Create your own copy of the project.
2.  **Create a Branch**: Work on your feature or bug fix in a new branch (e.g., `git checkout -b feature/new-chart`).
3.  **Make Your Changes**: Ensure your code follows the project's style and includes necessary tests.
4.  **Submit a Pull Request (PR)**:
    *   Target the `main` branch of the original repository.
    *   Provide a clear and concise description of your changes.
    *   Link to any related issues.

## The Review Process

*   **Approval Required**: All contributions must be reviewed and approved by the project maintainer before they are merged into the `main` branch.
*   **Discussion**: Maintainers may ask questions or suggest changes within the PR discussion.
*   **Data Safety**: **Never** include sensitive employee data or SQLite databases in your contributions.

## local Development

*   Follow the instructions in the [README.md](README.md) to set up your local environment.
*   Run tests before submitting: `pytest tests/`

Thank you for helping us build a more private way to analyze workforce data!
